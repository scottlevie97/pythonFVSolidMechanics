1. Introduction¶

Notebook 2

Welcome to this set of jupyter notebooks that outline how to build a finite volume solid mechanics solver. The notebooks contains descriptions of each stage of creating the code.

The solver produced from these notebooks does the following:

  • Uses the cell-centred finite volume method
  • Uses a structured mesh with rectangular geometry
  • Assumes a linear elastic solid model
  • Allows for fixed displacement or fixed traction boundary conditions

Problem Statement:¶

The solver produced by these notebooks will solve a solids mechanics problem. The problem is defined under the following four headings:

  • Geometry
  • Material Law
  • Boundary Conditions
  • Governing equations

The problems the solver will be able to solve will have some restrictions under the following headings:

Geometry: To keep this solver relatively simple the geometry of the problem will restricted to 2D dimensional space and a rectangular shape of solid material. This will be further outlined in this notebook.

Material Law: The solid will be assumed to be linear elastic. This means that all strains are assumed to be small. This material law defines the behaviour of the solid domain. This will be further outlined in this Notebook 02.

Boundary conditions: The conditions present at the boundary of the domains will be confined to either fixed displacement or fixed traction boundary conditions. This will be further outlined in this Notebook 04 and 05.

Governing equation: The problem will satisfy a governing equation: the conservation of momentum. This will be further outlined in this Notebook 02.

Example problem¶

In order to "test" the solver, we need an example test case. One that fits into the bracket of the above headings is a cantilever beam. The beam will be fixed at one end with an applied load at another end.

The solver should not be limited to just this test case. It should allow for any geometry and boundary conditions within the confinements above.

Geometry¶

The geometry of the spatial domain for this solver will be rectangular with an x-dimension length of $L_x$ metres and a y-dimension length of $L_y$ metres.

Defining Mesh¶

The spatial domain will be split into cells. Each cell will represent a volume of material where its behaviour will be described by a set of governing equations. The solver will compute what will happen at each cell. Each computational cell is rectangular with x-dimensions of $dx$ metres y-dimensions of $dy$ metres. This is what is meant by a structured mesh.

The number of cells in the $x$-direction = $n_x$

The number of cells in the $y$-direction = $n_y$

In [8]:
nx = 4  # Number of cells in the $x$-direction
ny = 4  # Number of cells in the $y$-direction

dy = Ly/ny  # Length of each control volume in the $y$-direction
dx = Lx/nx  # Length of each control volume in the $x$-direction

Displacement Fields¶

The displacement field $\boldsymbol{U} (x,y)$ can be thought of as the change in position of each cell center from its original position. This is not to be confused with coordinates $(x,y)$ which are the distances from the origin. Therefore we can initialise the $\boldsymbol{U}$ field to be equal to zero for all points in all directions.

As our mesh is 2D the $\boldsymbol{U} (x,y)$ (U) field is made up of change of displacement in the $x$-direction $u$ (u) and in the $y$-direction $v$ (v).

In [11]:
import numpy as np

# Initialise U field:


def initialise_U_field(nx, ny):

    # Create an zero array for the u and v values for each point
    U = np.zeros([nx*ny, 2])

    return U


U = initialise_U_field(4, 4)#

Note on Changing Displacement:¶

This solver will be used to find a displacement field that satisfies external conditions (loading, fixed displacement etc.). One assumption for this solver is that the solution displacement field has little effect on the geometry (volumes, face areas, etc.) of the mesh, i.e., it is assumed that these displacements are so small that the results from creating a new mesh including the new displacements would be very similar to the results without creating a new mesh. Hence, the mesh geometry will remain constant throughout the solution process. This is called the linear geometry approach, also known as the "small strain" or "small strain/rotation" approach.
